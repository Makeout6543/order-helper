# 下单助手 — 项目信息

## 访问地址

| 用途 | 地址 |
|------|------|
| 🏠 **日常使用**（唯一需要记的） | https://makeout6543.github.io/order-helper |
| 代码仓库 | https://github.com/makeout6543/order-helper |
| 后端 API（已写死，不用管） | https://order-helper.onrender.com |

## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | React + TypeScript + Tailwind CSS | 响应式、移动端优先 |
| PWA | vite-plugin-pwa | 离线可用、添加到主屏幕 |
| 构建 | Vite | 开发/构建一条龙 |
| 后端 | Express.js + tsx | TypeScript 直跑，无需编译 |
| 数据库 | SQLite (sql.js) | 纯 JS，文件型数据库 |
| 部署-前端 | GitHub Pages | 免费静态托管 |
| 部署-后端 | Render | 免费 Web Service |
| 防休眠 | 系统 cron 定时 ping | 每 10 分钟自动保活 |

## 本地开发

```bash
# 前端（端口 5173）
cd order-helper
npm run dev

# 后端（端口 3002）
cd order-helper/backend
npm run dev
```

开发时前端自动代理 `/api` 到 `localhost:3002`，无需手动改地址。

## 部署命令

```bash
# 有代码改动后重新部署前端：
npm run deploy
# 自动执行 → tsc 编译 → vite 构建 → 推到 GitHub Pages
# 后端：Render 自动监听 GitHub 仓库 main 分支变更，自动部署
```

## 添加到手机主屏幕（iPhone）

1. Safari 打开 https://makeout6543.github.io/order-helper
2. 点底部分享按钮 → 「添加到主屏幕」
3. 点「添加」

即可像原生 App 一样使用，全屏、离线可用。

## ⚠️ 注意事项

- 数据存在 Render 服务器本地 SQLite，**重启或重新部署会丢失**
- 暂无两台手机数据同步（纯单设备服务端存储）
- 后续如需数据持久化，可迁移到 Supabase 免费 PostgreSQL
